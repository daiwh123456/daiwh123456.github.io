JDBC

一、JDBC六大编程步骤

  1. 加载驱动 - 加载驱动实现类Driver
  2. 获取连接 - DriverManager
  3. 获取语句对象Statement/PreparedStatement
  4. 执行sql - 由语句对象将sql发送到mysql-server，如果执行的是DQL需要进行结果集处理，执行第五步
  5. 处理结果集
  6. 释放资源 - ResultSet/Statement/Connection

二、SQL注入

Statement语句对象可能会引发sql注入问题，因为参数硬拼接到了sql语句中

sql注入就是通过某种方式将sql代码添加到输入参数中，然后传递到sql服务器解析，比如说一条数据可能是select * from user where id = 1;但可能通过某些方式，可以将这个1改成 1 union select password from user,这样的语句同样会执行，容易造成数据的泄露等情况。

三、PreparedStatement和Statement区别

  • PreparedStatement的效率比Statement高
  • PreparedStatement继承于Statement，都是接口
  • Statement会造成SQL注入，PreparedStatement有效防止sql注入
  • PreparedStatement会预编译SQL语句，Statement每次都会解析编译SQL
  • PreparedStatment适用于执行同构的sql语句，在运行时接受输入的参数。Statement适用于静态SQL语句，在运行时不接受输入的参数

四、连接池的工作原理

数据库连接池的基本思想就是为数据库建立一个“缓冲池”。预先在缓冲池中放入一定数量的连接，当需要建立数据库连接时，只需要从“缓冲池”中取出一个，使用完毕之后再放回去。

连接池的工作原理主要由三部分组成，分别为连接池的创建，连接池中连接的使用管理，连接池的关闭。

  • 连接池的创建

    一般系统初始化的时候，连接池会根据系统的配置来建立，并且在池中创建指定数量的连接对象，以便使用时能从连接池中获取。

    连接池的连接不能随意创建和关闭，这样避免了连接随意建立和关闭造成的系统开销。

  • 连接池连接的使用策略

    客户请求数据库连接时，首先判断连接池中是否有空闲的连接，如果存在空闲的连接，则直接将连接分配给客户使用。如果没有空闲的连接，就需要判断当前所开的连接数是否已经达到了最大的连接数，如果没有达到就重新创建一个新的连接给请求的客户。如果已经达到了最大的连接数就需要按照设置的最大等待时间进行等待，如果超出了最大的等待时间，则抛出异常给客户。

    客户释放连接时，先判断该连接的引用次数是否超过了规定值，如果超过就从连接池中删除该连接，否则就保留为其他客户服务。

    该策略保证了数据库连接的有效复用，避免频繁的创建或释放连接所带来的系统资源开销

  • 连接池的关闭

    当应用程序退出时，关闭连接池中所有的连接，释放连接池相关的资源，该过程正好与创建相反。

五、连接池相关配置

  • maxActive - 连接池支持的最大连接数
  • maxIdle - 连接池中最多空闲连接个数
  • minIdle - 连接池中最少空闲连接个数
  • initialSize - 初始化连接数目
  • maxWait - 连接池中连接用完时新的请求等待时间

Mybatis

六、ORM思想

概念：对象关系映射（Object Relational Mapping，简称ORM）模式是一种为了解决面向对象与关系数据库存在的互不匹配的现象的技术。ORM是通过使用描述对象和数据库之间映射的元数据，将程序中的对象自动持久化到关系数据库中.

ORM提供了实现持久化层的另一种模式，它采用映射元数据来描述对象关系的映射，使得ORM中间件能在任何一个应用的业务逻辑层和数据库层之间充当桥梁。

Java典型的ORM中间件有:Hibernate,Mybatis等。

对象关系映射，表和实体类一一对应。

O - Object - 对象

R - Relational - 关系

M - Mapping - 映射

七、Hibernate和Mybatis的区别

  • Mybatis可以优化SQL语句，Hibernate不能。

  • Hibernate是基于ORM思想开发的全自动框架，Mybatis是基于ORM思想开发的半自动框架。

  • Hibernate开发比MyBatis更简单，Mybatis需要维护SQL和结果映射。

  • Hibernate移植性很好，MyBatis移植性不好，不同的数据库需要写不同的SQL语句。

八、Mybatis(面对简单的封装,自动封装结果集)和JDBC(手动封装结果集)区别

  1. Mybatis内部封装了JDBC，开发只关注SQL语句本身
  2. MyBatis优化了获取和释放，避免了加载驱动，创建连接等繁琐的过程
  3. MyBatis可以通过标签动态的生成SQL语句
  4. MyBatis可以对结果集进行映射

九、${}和#()区别

  1. #{}是预编译处理，${}是字符串替换
  2. 使用#{}可以有效的防止sql注入，提高系统安全性
  3. MyBatis在处理#{}时会将sql中的#{}替换成?号，调用PreparedStatement的set来赋值
  4. MyBatis在处理${}时就是把${}替换成变量的值

十、ResultType指定返回类型和ResultMap区别

resultType:

  1. 只有select标签才需要指定resultType属性————用来指定查询出来的返回值类型

  2. 当查询出来的列名和实体类属性名高度保持一致，能够实现自动映射

或者出来的列名是一个合法的匈牙利命名 - 实体类属性的小驼峰.

resultMap:

  1. 当查询的列值不能自动映射

  2. 需要通过resultMap来进行实体类属性的一一绑定.

十一、selectOne和selectList区别

  1. selectOne：是用来接收一条结果集
  2. selectList：是用来接收多条结果集
  3. 如果返回的数据不只一条，使用selectOne来接收，那么会抛出异常

十二、延迟加载

什么是延迟加载
  1. 也称懒加载
  2. 需要数据时才进行加载，不需要就不加载
  3. 在关联查询时，先加载主信息，使用到关联信息时再去加载关联信息
好处
  1. 首次只加载主要信息，关联信息等用户需要时再加载
  2. 减少数据库压力，提高数据库性能
坏处
  1. 因为只有需要数据时，才会进行数据库查询
  2. 这样在大批量数据查询时，因为查询工作也要消耗时间，可能造成用户等待时间变长，体验下降
PS
  1. Mybatis 仅支持 association 关联对象和 collection 关联集合对象的延迟加载
  2. association指的就是一对一，collection 指的就是一对多查询

十三、一级缓存和二级缓存

一级缓存
  1. Mybatis默认开启一级缓存
  2. 一级缓存是SqlSession级别的缓存，作用域是SqlSession范围
  3. 第一次执行完毕会将查询数据库中的数据写到缓存（内存），第二次査询时会从缓存中获取数据，不再去底层数据库查询，从而提高查询效率
  4. 优先到一级缓存中去查询数据，如果有则返回，没有则进行DB交互
二级缓存
  1. 二级缓存是mapper级别的缓存，作用域是mapper的同一个namespace
  2. 二级缓存需要手动配置启动
  3. 优先到一级缓存中去查找是否存在对象，如果存在返回，不存在，再到二级缓存中去查找，存在返回，不存在进行DB交互
  4. 将对象散列的属性数据放一份到二级缓存，将整个实体对象放入到一级缓存

一级缓存: 是SqlSession级别的缓存，在操作数据库时需要构造SqlSession对象，在对象中有一个HashMap用于存储缓存数据，不同的SqlSession之间缓存数据区域（HashMap）是互相不影响的。

如果SqlSession执行了DML操作（insert、update、delete），并执行commit（）操作，mybatis则会清空SqlSession中的一级缓存，这样做的目的是为了保证缓存数据中存储的是最新的信息，避免出现脏读现象。

Mybatis默认开启一级缓存，不需要进行任何配置。

二级缓存: mapper级别的缓存，二级缓存是多个SqlSession共享的。

它同样是使用HashMap进行数据存储，相比一级缓存SqlSession，二级缓存的范围更大，多个SqlSession可以共用二级缓存，二级缓存是跨SqlSession的。

mybatis通过缓存机制减轻数据库压力，提高查询性能。

Mybatis默认没有开启二级缓存，需要在setting全局参数中配置开启二级缓存。

    pageHelper

    Mybatis使用RowBounds对象进行分页，它是针对ResultSet结果集执行的内存分页，而非物理分页，可以在sql内直接书写带有物理分页的参数来完成物理分页功能，也可以使用分页插件来完成物理分页。

    ​ 分页插件的基本原理是使用Mybatis提供的插件接口，实现自定义插件，在插件的拦截方法内拦截待执行的sql，然后重写sql，根据dialect方言，添加对应的物理分页语句和物理分页参数。
